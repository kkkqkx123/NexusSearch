# FlexSearch v0.8.2 - 项目概述

FlexSearch 是一个下一代全文搜索引擎，支持浏览器和 Node.js 环境。本项目包含原始的 JavaScript 库实现以及基于 Go + Rust 混合架构的中间层服务实现，提供高性能的搜索能力。

## 项目架构

本项目采用**混合架构设计**，包含两个主要部分：

### 1. 原始 FlexSearch 库 (JavaScript)
- 核心搜索引擎库，支持多种构建配置
- 提供 Index、Document、Worker 等多种索引类型
- 支持模糊搜索、上下文搜索、标签搜索、结果高亮等功能

### 2. 中间层服务 (Go + Rust)
采用 Go + Rust 混合架构，充分发挥两种语言优势：
- **Go**: 用于网络服务和协调层（API 网关、查询协调器）
- **Rust**: 用于高性能计算密集型任务（搜索引擎核心）

## 核心功能

1. **API 网关**
   - JWT Token 验证和 API Key 认证
   - 基于用户/IP 的限流控制
   - 请求路由和格式转换
   - 统一错误处理

2. **查询协调器**
   - 智能查询路由
   - 并行调用多个搜索引擎
   - RRF/加权结果融合
   - Redis 查询缓存
   - 超时控制和降级机制

3. **Inversearch 服务**
   - 高性能倒排索引
   - 精确/模糊/短语匹配
   - 结果高亮
   - 搜索建议

4. **BM25 服务**
   - 基于 BM25 算法的全文搜索
   - 词频统计和文档频率统计
   - k1 和 b 参数调优
   - 多语言分词支持

## 技术栈

### Go 服务
- Go 1.23+
- Gin (HTTP 框架)
- gRPC (服务间通信)
- Redis (缓存/限流)
- Zap (日志)
- Prometheus (监控)
- Viper (配置管理)

### Rust 服务
- Rust 2021
- Tokio (异步运行时)
- Tonic (gRPC 框架)
- Tantivy (搜索引擎)
- Redis (缓存)
- Tracing (日志)
- Metrics (监控)

## API 概览

### 主要方法
- `add(id, content)` - 添加内容到索引
- `search(query, options)` - 执行搜索
- `update(id, content)` - 更新索引内容
- `remove(id)` - 从索引中删除
- `clear()` - 清空整个索引
